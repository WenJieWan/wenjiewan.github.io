Java集合知识总结

wwj 发表于 2017-05-27


本章是对java中集合相关知识的总结

Map概览

在 Java SE 中有4个Map常用的实现，

分别是HashMap，TreeMap，HashTable和LinkedHashMap。

HashMap的底层的数据结构是哈希表，它中的键值是 无序的 ， 线程不安全 。

Hashtable是同步的，与HashMap相比它有个同步的开销，因此是 线程安全的 。

LinkedHashMap是HashMap的子类，集成了它的特点并 保留了插入的顺序** ,使用双向链表维护元素的次序**。

TreeMap是基于红黑树结构的一个实现，实现了SortedMap接口，它会根据key的自然顺序进行排序，或者根据创建映射时提供的 Comparator 进行排序， 是有序的 。

ConcurrentHashMap 的底层的数据结构是哈希表，无序，线程安全，性能好于Hashtable,适用于高并发。

Map注意点

HashMap : 初始化大小16，负载因子默认0.75 ,阀值为容量大小*负载因子 （JDK1.7）

初始化大小16，负载因子默认0.75,阀值为容量大小*负载因子，添加元素超过8个或者初始化大小超过64个，链表将转化成红黑树结构保存 （JDK1.8）

当元素个数达到或超过负载因子则长度翻倍并重新hash分布（JDK1.7,JDK1.8）

Hashtable : 初始化大小11，负载因子默认0.75 ,put,get,remove… 等相关操作都加同步锁Synchronized关键字来修饰。 （JDK1.7,JDK1.8）

LinkedHashMap :继承自HashMap,与其一样，多了头尾指针的变量。 （JDK1.7,JDK1.8）

TreeMap :初始化大小为0，内部实现Comparator。 （JDK1.7,JDK1.8）

ConcurrentHashMap :初始化大小为16，采用分段锁技术实现，利用多个segement来一对一的进行维护多个哈希表（类似hashmap的结构）。 （JDK1.7）

初始化大小为16，采用CAS技术+volatile实现,链表转换跟1.8的hashmap一样。 （JDK1.8）

Set概览

HashSet的底层的数据结构是哈希表， 无序，高效** ， **线程不安全 。

LinkedHashSet是HashSet的子类，集成了它的特点并 保留了插入的顺序** ,使用双向链表维护元素的次序**。

TreeSet是基于红黑树结构的一个实现，实现了SortedSet接口，它会根据key来 自动** 排序 ， 是有序的**。

Set注意点：

HashSet是内部维护的是一个Hashmap的对象

TreeSet是内部维护的是一个Treemap的对象

LinkedSet是链表

List概览

ArrayList的底层的数据结构是 数组 ， 有序，查找快** ， **线程不安全 。

LinkedList的底层的数据结构是 链表，有序，增删快** ， **线程不安全 。

Vector的底层的数据结构是 数组 ， 有序，查找增删都巨慢** ， **线程安全 。

List注意点：

ArrayList是按照原数组的50%延长，初始化大小为10

Vector是按照原数组的100%延长，初始化大小为10

LinkedList初始化大小为0
